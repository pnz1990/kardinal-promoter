## Context

kardinal-ui is an embedded React 19 + TypeScript UI served by the controller binary at `/ui`. It renders the promotion DAG with per-node state (PromotionStep: green/amber/red; PolicyGate: pass/fail/pending). Read-only. Built with Vite, bundled via `go:embed`.

---

## User Scenarios & Testing

### User Story 1 — Pipeline DAG visible in browser (Priority: P1)

A user opens `http://controller:8080/ui` and sees all Pipelines with a DAG view showing PromotionStep and PolicyGate node states.

**Why this priority**: Operators need to see live promotion status without kubectl.

**Independent Test**: Vite unit tests + Go HTTP handler test. `go test ./web/... && cd web && bun test`

**Acceptance Scenarios**:

1. **Given** a Pipeline with 3 environments and 2 PolicyGates, **When** user visits `/ui`, **Then** the DAG renders with 3 PromotionStep nodes and 2 PolicyGate nodes in correct topology
2. **Given** a Verified PromotionStep, **When** user views the DAG, **Then** the node is green
3. **Given** a failed PolicyGate, **When** user views the DAG, **Then** the node is red and the CEL expression + current value are shown in the node detail panel

---

### User Story 2 — PR link visible from DAG node (Priority: P1)

Clicking a WaitingForMerge PromotionStep node shows the PR URL as a clickable link.

**Acceptance Scenarios**:

1. **Given** a PromotionStep with `status.prURL` set, **When** user clicks the node, **Then** the detail panel shows a clickable PR link
2. **Given** a PromotionStep with `status.evidence.approvedBy`, **When** user clicks the node, **Then** the detail panel shows who approved

---

### User Story 3 — Port separation for security (Priority: P2)

When `--ui-listen-address=:8081` is set, the UI is on port 8081 and the API/webhooks stay on 8080.

**Acceptance Scenarios**:

1. **Given** `--ui-listen-address=:8081`, **When** the controller starts, **Then** `/ui` is served on 8081, `/api/v1/bundles` is on 8080

## Requirements

- **FR-001**: React 19 + TypeScript + Vite, bundled into `web/dist/`, embedded via `go:embed`
- **FR-002**: Backend API proxy at `/api/v1/ui/` reading CRDs from K8s API server
- **FR-003**: DAG rendered with dagre for layout; PromotionStep = rounded rect, PolicyGate = hexagon
- **FR-004**: Node states color-coded (Verified=green, Promoting=amber, Failed=red, Pending=gray)
- **FR-005**: Read-only — no mutation from UI
- **FR-006**: Data refreshed every 5 seconds (polling in Phase 1)
- **FR-007**: `--ui-listen-address` flag for port separation

### Package Structure

```
web/
  embed.go        # go:embed all:dist
  src/
    pages/        # PipelineList, PipelineDetail, BundleList
    components/   # DAGGraph, DAGNode, NodeDetail, PolicyGateCard
    lib/          # api.ts, types.ts, dag.ts
  dist/           # built assets (gitignored, generated by bun run build)
```
